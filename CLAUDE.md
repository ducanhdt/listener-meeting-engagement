# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A research project (CCS2) that analyzes participant **engagement / nonverbal behavior** in recorded multi-party meetings (the GEHM corpus). Speaking turns are cut into 5-second video clips, behavioral features are extracted per clip, and clips are clustered to discover behavior types and compared against human engagement labels.

Code lives in Jupyter notebooks. There is no package, build step, or test suite.

**Running locally vs. Colab.** The four original pipeline notebooks were written for **Google Colab** — they start with `drive.mount('/content/drive')` and read/write under `/content/drive/MyDrive/data/`. To run those locally you must rewrite those paths to point at this repo's root, where the produced `.pickle`/`.csv`/`.zip` artifacts already live. **`additional_features.ipynb` is the local-first notebook** (see below) — run it from the repo root with no path edits.

## Pipeline (run notebooks in this order)

1. **CCS2_data_processing.ipynb** — Raw → clips + features. Downloads meeting recordings, splits per-speaker video by speaking segments into 5s clips, then extracts:
   - OpenPose keypoints (`read_keypoint_folder`) and head orientation (yaw/pitch/roll via `estimate_head_orientation`) → "manual" behavior features (nods, shakes, chin rest, face touch, hand raise, movement velocities).
   - Facial Action Units (`AU*` columns).
   - DNN visual embeddings via OpenAI CLIP. Outputs `behavior_results.zip` / `reference_behavior_results.zip`, `dnn_feature.pickle`, `dnn_feature_reference.pickle`, and segmentation CSVs.

2. **feature_gathering.ipynb** — Features → one table. Joins manual features, AU features, and DNN features per segment. Key transforms: `preprocess_manual_features(row, features)` builds a numeric vector (one-hot for categoricals, 0/1 for booleans) from the hardcoded `features` list; `process_au_features(row)` concatenates all `AU*` floats. Computes Fleiss' kappa on annotator labels. Produces **`all_features_combined.pickle`** (also `.csv`) and **`label_df_3.pickle`** (also `.csv`).

3. **clustering.ipynb** — Clustering + evaluation. Loads the two pickles, PCAs the DNN and AU features, splits labeled segments (`label_df_3`) out as a test set, runs **KMeans, GMM, and Agglomerative** clustering, and exports representative clips per cluster into `kmean/`, `GMM/`, `HIE/` (one subfolder per cluster index). Then maps clusters → engagement labels and evaluates with a RandomForest classifier + ANOVA F-values for feature importance.

4. **clustering_alignment.ipynb** — Same flow as `clustering.ipynb` but adds an **alignment feature**: cosine similarity between a participant and their reference person(s). Outputs go under `reference_cluster/`. (Already uses local `./` paths; its cluster→engagement maps in the "Manually assign label" cell are hardcoded from manual clip inspection of *that* clustering — they do not transfer if you change the feature set.)

5. **clustering_alignment_extend.ipynb** — `clustering_alignment.ipynb` re-run on `all_features_combined_extended.pickle` (alignment features **+ speaker features**). Builds `X_baseline` (alignment-only) vs `X_extended` (+ speaker block: median-imputed `spk_ac_*`, zero `spk_txt_*`, `spk_has_speaker` flag; categorical `speaker_id` dropped), auto-picks cluster k by silhouette/BIC, replaces the hardcoded cluster→label maps with an **automatic majority-vote** map, and adds the headline **cross-validated baseline-vs-extended** comparison (RandomForest + LogisticRegression macro-F1) plus RF/ANOVA importance and a written **Findings** cell. Result: speaker features give a small RF gain (macro-F1 0.231→0.262); the most informative speaker cue is pitch variability (`spk_ac_f0_std`); engagement prediction stays hard (~89 labeled segments). Run in the `thesis2` conda env (has parselmouth/VADER).

### Unified analysis: `utils.py` + `analysis.ipynb` (supersedes notebooks 3–5)

Steps 3–5 above were near-duplicate notebooks (~7 verbatim-identical functions plus the same load/PCA/split/cluster/CV/importance blocks) that differed only in *which* feature set they stacked. They are now consolidated:

- **`utils.py`** (repo root, local-first) holds every shared function: `load_features`/`load_labels`, `add_dnn_pca`/`add_au_pca` (PCA dims 100/20), `add_alignment_features` (+`calculate_cosine_similarity`), `split_train_test`, `extract_and_combine_features`, `fit_speaker_imputer`/`speaker_block` (train-median imputation, no leakage), `segment_id_to_filepath`/`clip_path`, `export_representative_clips`, **cluster-count selection** (`silhouette_by_k`/`select_cluster_counts`/`gmm_bic_aic`/`plot_dendrogram`) + `fit_clusterers`, the cross-validated comparison (`make_cv`/`cv_report`/`kmeans_ari`/`rf`/`lr`), the compact Head-and-Gesture encodings (`make_hg_blocks`, `HG_GROUPS`), and feature interpretation (`plot_feature_importances`/`engagement_feature_importance`, readable names via `MANUAL_FEATURE_NAMES`/`prettify_feature_names`, family aggregation via `feature_family`/`family_importance`). Column-group constants `TARGET_ONLY_COLS`, `ALIGN_COLS`, `COSINE_ALIGN_COLS`, `LS_ALIGN_COLS` live here too. **Feature-importance caveat:** RF *impurity* importance is dim-count biased (CLIP's ~100 dims take 0.63; the listener↔speaker block — best in the CV ablation — ranks near the bottom at 0.01) and permutation importance is ≈0 for every feature at n=89 → no single feature is robustly pivotal. Trust the **CV ablation (Table 1)** for what helps; the importance charts are descriptive only. The **terminal pitch slope** (`spk_ac_final_rise_st`) per-class means (Active +1.48 / Low +2.63 / Disengaged −3.58 **st/s**) are a robust *descriptive correlate*, not a standalone predictor. (Unit is semitones **per second**, not semitones: `final_rise_st` is `polyfit(time, semitones, 1)[0]`, the slope of median-relative pitch — `st = 12·log2(f0/median_f0)` — over the final 0.5 s of voiced speech. Earlier notes that wrote "st" were imprecise.) **Note:** the old in-sample `auto_cluster_label_map`/`print_evaluation`/`evaluate_clusters` cluster→engagement scoring was **removed** — it was optimistic (named clusters with the same test labels it scored against) and collapsed to the majority class for degenerate clusterings (e.g. GMM put all 89 labeled points in one component). The honest unsupervised signal is `kmeans_ari` (KMeans k=3 ARI ≈ −0.01 to −0.03 → clusters do not recover engagement); the rigorous predictive answer is the cross-validated table.
- **Listener↔speaker alignment (`add_listener_speaker_alignment` / `LS_ALIGN_COLS` / `fit_ls_imputer` / `ls_align_block` in `utils.py`; `use_ls_align` flag, extended only).** Distinct from the existing listener↔*reference* alignment: it measures similarity between the silent listener and the **identified active speaker** (`speaker_id`) in the same window, on the visual (CLIP) and AU channels (cosine + correlation → 4 scalars). The speaker's features are looked up from already-extracted assets — `dnn_feature_reference.pickle` + `reference_behavior_results.zip` — so **no new downloads**; ~64% speaker coverage, rest train-median imputed (no leakage). **Result (reproducible from `analysis.ipynb`):** it is the single best feature addition for the 3-class target (RF macro-F1 0.341 → **0.378**), above listener↔others alignment (0.326) and the speaker block (+speaker 0.364); the *relation* beats either party's features alone, though the gain is within fold std at n=89. This is the corrected, in-pipeline version of the prototype (was 0.384 with full-set-median imputation; the train-median fix is the honest 0.378).
- **Default alignment = the 6 cosine-similarity scalars; `manual_diff_*` is dropped.** The 138-dim element-wise `manual_diff_*` difference block overfits the ~89 labels and *hurts* prediction (RF macro-F1 0.231 with it vs 0.326–0.364 without), and it encodes dissimilarity. The notebook sets `ALIGN_USE = COSINE_ALIGN_COLS` (6 cosine sims); `add_alignment_features` still computes all 9 columns so the full `ALIGN_COLS` block is available only for the `[ref]` comparison rows that demonstrate the collapse. Optional Head-and-Gesture cosine (`make_hg_blocks`, 3-d) is a further compact alignment add-on.
- **Cluster counts: `select_cluster_counts` is a diagnostic, `k_override` decides.** `select_cluster_counts` scores KMeans/GMM/Agglomerative by **silhouette** over a `k` range (one consistent, label-agnostic criterion) with `gmm_bic_aic` as a cross-check, but structure is weak — silhouette/BIC prefer **k=2** and clusters never recover engagement (`kmeans_ari` ≈ −0.01 to −0.03 at any k). Because cluster count is unrelated to the 3 engagement classes and clustering is exploratory here, `analysis.ipynb` sets **`k_override = dict(kmeans=3, gmm=3, agg=3)`** as a *presentation* choice (comparable to the 3 classes); set `k_override={}` to use the silhouette-optimal k=2 instead.
- **Engagement target is regroupable via `LABEL_SCHEME` (`apply_label_scheme` / `LABEL_SCHEMES` in `utils.py`).** Default/primary is **`3class`** (the annotation `majority_label_3`, with the raw middle `Intermittent engagement` displayed as **`Low engagement`** — an annotator hedge for "unsure"). Three reductions are reported as a **sensitivity analysis** (final notebook cell, all schemes × feature sets in one table): `active_vs_rest` (Active vs Low+Disengaged; "is the listener actively engaged?") is balanced ~44/45 and **learnable** (RF macro-F1 ≈ 0.63 vs 0.34 for `3class`) but the signal is the **listener's own** behaviour; `engaged_vs_disengaged` (Active+Low vs Disengaged) is 81/19 and the 17 disengaged are unlearnable (collapses, disengaged F1 = 0); `confident_only` (Active vs Disengaged, the uncertain middle **dropped → NaN, rows excluded**, n=61) still fails (≈ floor). All three say the binding constraint is **disengaged labels**. Clustering is unsupervised so it is unaffected by the scheme.
- **`analysis.ipynb`** is the thin driver. Set **`CONFIG`** (`"base"`/`"alignment"`/`"extended"`) and **`LABEL_SCHEME`** at the top; a `SETTINGS` dict selects the input pickle, alignment/speaker blocks, the output folder, and `k_override`. Functionally `"base"` = old `clustering.ipynb`, `"alignment"` = `clustering_alignment.ipynb`, `"extended"` = `clustering_alignment_extend.ipynb` (the CV headline + manual_diff-vs-cosine ablation + speaker-importance cells only run for alignment/extended).
- **The feature story depends on the label scheme — report it per scheme:**
  - `3class` (extended): full factorial over add-on blocks {others-align O, listener↔speaker LS, speaker S} on top of target-only (0.341). **Best single addition is LS alone: 0.378**; others-align 0.326, speaker 0.306. **Blocks do not stack** — every richer combo dilutes (O+S 0.364, LS+S 0.368, all O+LS+S 0.337 ≈ baseline): added dims cost more than the signal at n=89. Dropped `[ref]` manual_diff 0.231. All within fold std (≈0.09) — directional, not significant.
  - `active_vs_rest` (extended): **target-only is already strongest** (RF macro-F1 **0.634**, +0.30 over the ~0.506 floor); adding alignment/speaker *does not help* (slightly hurts), because "Active vs not-actively-engaged" is driven by the listener's own nonverbal features, not the interlocutor. So under `active_vs_rest` the honest headline is "the listener's own behaviour predicts active engagement; alignment/speaker add nothing" — the opposite of the (weak) 3-class direction.
  - Both schemes: dropped `[ref]` manual_diff stays worse than the compact cosine encodings; cluster ARI ≈ 0.
- The three original notebooks are **kept for reference** but are superseded by `analysis.ipynb`. Run it from the repo root in the `thesis2` conda env (clustering itself needs only pandas/numpy/sklearn/scipy/matplotlib/seaborn — no parselmouth/VADER).

### Adding new features without reprocessing

**additional_features.ipynb** (local-first; run from the repo root) is the entry point for computing *extra* per-segment features while **reusing** everything already processed. It:
- unzips `behavior_results.zip` once and loads `all_features_combined.pickle`, `dnn_feature.pickle`, `label_df_3.csv`, `target_segmentation.csv` — no recomputation of existing features;
- provides segment-ID helpers (`parse_segment_id`, `behavior_csv_path`, `dnn_key`, `load_behavior_row`) that map a `segment_id` to its already-extracted raw behavior CSV and CLIP embedding;
- downloads raw GEHM assets **only when needed** via `download_raw_for_meeting(meeting, persons, what=("video","keypoints","textgrid","audio"))`, cached under `raw_data/<meeting>/` and skipped if present (source: `https://archive.org/download/GEHM_meeting_corpus/corpus/<meeting>/...`; `audio` = per-speaker `.flac` from `SeparatedSpeakersAudio/`);
- implements the new feature in `compute_extra_feature(segment_id)` and writes the result to **`all_features_combined_extended.pickle`/.csv`** — the originals are never overwritten.

To cluster with a new feature, point `clustering.ipynb` / `clustering_alignment.ipynb` at `all_features_combined_extended.pickle` and add the new column to the stacked feature set.

#### Speaker-side features (section 6 of `additional_features.ipynb`)

Every target/listener segment is, by construction, a window where the target is *not* speaking. To predict the listener's engagement, that section characterizes the **person speaking to them** during the same window:
- `find_speaker(segment_id)` → `(speaker_person_id, overlap_seconds)`: loads the meeting TextGrid (each participant has a word-level tier named by person_id), and picks the *other* participant whose tier has the most speech overlapping the window; `(None, 0.0)` on silence. Spurious `[-1,-1]` marker intervals in the TextGrids are filtered.
- `compute_speaker_acoustic_features(segment_id)` — **event-based** prosody + voice quality from the speaker's separated `.flac` (sliced to the window) via **Praat/parselmouth**, with voiced F0 in **semitones relative to the window median** (speaker-independent): `f0_mean`, `f0_std_st`, `pitch_rise_count`/`_rate`, `pitch_fall_count`, **`final_rise_st`** (terminal pitch slope — the strongest engagement cue), `f0_slope_st`, `f0_curv`, `accent_count`, `intensity_mean`/`_slope`/`_peak_count`, `pause_count`/`pause_ratio`, `voiced_rate`, `articulation_rate`, `jitter`, `shimmer`, `hnr`. (Transcripts have no punctuation, so question/engagement intonation must come from pitch, not `?`.)
- `compute_speaker_text_features(segment_id)` — **interactional** English lexicon features: `word_count`, `words_per_sec`, `question_cue_count` + `starts_question`, `backchannel_count`/`_rate`, `filler_rate`, `negation_rate`, second-/first-plural-person rates, **VADER** `sent_compound`.
- `compute_speaker_context_features(segment_id)` — turn-taking context from the TextGrid (`spk_ctx_*`): `num_speakers`, `speaker_lead_sec`, `listener_recent_sec`, `meeting_position`, `overlap_others`.
- The batch cell composes prefixed columns (`speaker_id`, `spk_ac_*`, `spk_txt_*`, `spk_ctx_*`) onto `all_features_combined_extended.pickle`. Extra deps: `praat-parselmouth`, `vaderSentiment`, `soundfile`, `scipy`.
- **Result (see `clustering_alignment_extend.ipynb` Findings):** speaker features lift RandomForest engagement macro-F1 from 0.231 (alignment-only) to ~0.30; **the signal saturates at pitch** — voice-quality and turn-taking context add nothing (context slightly hurts). Top cue is **terminal pitch slope** `final_rise_st` (disengaged listeners get falling/flat intonation ≈ −3.6 st/s vs rising for engaged; unit is semitones **per second** — slope over the final 0.5 s). A **compact 5-feature pitch set** (`f0_mean`, `final_rise_st`, `f0_std_st`, `f0_slope_st`, `sent_compound`) scores macro-F1 0.350 *alone* — ≈ the full 643-dim listener feature set (0.373) — but is largely redundant on top of it given only ~89 labels. Signal is non-linear (helps trees, not logistic regression). A keypoint-based **speaker↔listener motion coupling** feature (Tier 1) was scoped but skipped for download cost.

## Key data artifacts (repo root)

- `all_features_combined.pickle` / `.csv` — the master per-segment feature table consumed by both clustering notebooks.
- `label_df_3.pickle` / `.csv` — human engagement annotations; columns `meeting, participant, video_clip, <annotator names>, segment_id, agreement, majority_label_3, vote_agreement`. `majority_label_3` is the ground-truth engagement class.
- `target_segmentation.csv` / `reference_segmentation.csv` — clip boundaries (`start_time`/`end_time`) and target↔reference person mapping.
- `kmean/`, `GMM/`, `HIE/`, `reference_cluster/` — clustering output: representative `clip_<start>_<end>.mp4` files grouped by cluster.

## Conventions

- **Segment IDs** have the form `<meeting_id>-<person_id>_clip_<start>_<end>` (e.g. `20211007-SP07F_clip_2652_2657`). `segment_id_to_filepath()` (in the clustering notebooks) converts these to clip paths — it builds Colab paths, so update it when running elsewhere.
- `meeting_to_participants` / `meeting_to_person` dicts at the top of cells enumerate which speakers to process per meeting date; edit these to add/restrict data.
- Person IDs encode gender as a suffix (`F`/`M`), e.g. `SP07F`.
- The hardcoded `features` list in `feature_gathering.ipynb` defines the manual feature vector layout; changing it changes vector dimensionality downstream, so keep it in sync if you regenerate `all_features_combined`.
- The final number of clusters is set manually per algorithm (`final_k_kmeans`, `final_n_gmm`, `final_k_hierarchical`) after inspecting silhouette/BIC/dendrogram plots — these differ between `clustering.ipynb` and `clustering_alignment.ipynb`.
